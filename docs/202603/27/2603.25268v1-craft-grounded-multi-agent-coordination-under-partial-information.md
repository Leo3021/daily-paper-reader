## 摘要
我们介绍了 CRAFT，这是一个用于评估大语言模型在严格部分信息下语用通信（pragmatic communication）的多智能体基准测试。在该设定中，多个拥有互补但非完整视野的智能体必须通过自然语言进行协作，以构建一个任何单个智能体都无法完全观察到的共享 3D 结构。我们将该问题形式化为多发送者语用推理任务，并提供了一个诊断框架，将失败原因分解为空间定位（spatial grounding）、信念建模（belief modeling）和语用通信错误，同时对前沿模型和开源权重模型中的行为失败概况进行了分类。通过对包括 8 个开源权重模型和 7 个前沿模型（含推理模型）在内的多种模型进行测试，我们发现更强的推理能力并不一定能可靠地转化为更好的协作能力：较小的开源权重模型往往能达到甚至超过前沿系统的表现，且个体通信能力的提升并不保证协作的成功。这些结果表明，多智能体协作对于当前的语言模型来说仍然是一个从根本上尚未解决的挑战。我们的代码可在 https://github.com/csu-signal/CRAFT 找到。

## Abstract
We introduce CRAFT, a multi-agent benchmark for evaluating pragmatic communication in large language models under strict partial information. In this setting, multiple agents with complementary but incomplete views must coordinate through natural language to construct a shared 3D structure that no single agent can fully observe. We formalize this problem as a multi-sender pragmatic reasoning task and provide a diagnostic framework that decomposes failures into spatial grounding, belief modeling and pragmatic communication errors, including a taxonomy of behavioral failure profiles in both frontier and open-weight models. Across a diverse set of models, including 8 open-weight and 7 frontier including reasoning models, we find that stronger reasoning ability does not reliably translate to better coordination: smaller open-weight models often match or outperform frontier systems, and improved individual communication does not guarantee successful collaboration. These results suggest that multi-agent coordination remains a fundamentally unsolved challenge for current language models. Our code can be found at https://github.com/csu-signal/CRAFT

#### 2. 方法论：CRAFT 框架与多发送者 BPS 理论
*   **核心思想**：构建一个名为 **CRAFT**（协作推理建筑任务）的基准，模拟 3D 结构搭建。
*   **关键角色**：

    *   **建筑者（Builder）**：无法看到目标，仅根据指令者的自然语言描述在 3x3 网格上放置或移除方块。
*   **技术细节**：
    *   **多发送者有限语用说话者（BPS）框架**：将协作形式化为数学模型，要求每个指令者不仅要考虑自己的私有观察，还要建立“联合听者模型”，预测其信息对建筑者的效用，从而实现 Gricean 语用准则（信息量适中、准确、相关）。
    *   **思维链（CoT）结构**：指令者输出分为私有推理（`<think>`）和公共消息（`<message>`），强制模型进行内部空间推理后再进行外部沟通。
    *   **Oracle 辅助接口**：为建筑者提供经过验证的候选动作，以消除建筑者执行能力的干扰，确保失败能准确归因于指令者的沟通质量。

#### 6. 主要结论与发现
*   **能力错位**：前沿模型（如 GPT-4.1-Mini, Claude）在个体推理和沟通质量评分上很高，但**实际协作任务进度反而不如小型开源模型**（如 Mistral-7B, Qwen-7B）。
*   **纠错螺旋（Correction Spirals）**：高性能模型容易陷入过度纠错的死循环，频繁要求建筑者移除方块以追求完美对齐，导致消耗大量轮次却无实质进展。
*   **个体能力 $\neq$ 团队成功**：更强的空间推理和视角切换能力并不直接转化为更好的协作效果。
*   **瓶颈所在**：3D 层的推理（Layer Inference）是所有模型共同的薄弱环节。

#### 7. 优点与亮点
*   **理论扎实**：将语用学理论（BPS）成功引入多智能体 LLM 协作分析。
*   **诊断深入**：不仅看任务是否成功，还通过三个维度的裁判模型拆解了“为什么失败”。
*   **解耦设计**：通过 Oracle 辅助建筑者，解决了多智能体系统中长期存在的“信用分配（Credit Assignment）”难题。

#### 8. 不足与局限
*   **模态限制**：目前仅限于纯文本环境，未测试多模态模型（VLM）在直接视觉观察下的表现。
*   **规模有限**：测试的 3D 结构数量（20个）相对较少，可能存在一定的随机性。
*   **同质化设置**：实验中所有指令者均使用同一模型，未探讨异构智能体（如 GPT 与 Llama 混合）的协作动态。
*   **静态建筑者**：建筑者角色被简化为选择器，缺乏主动探索或纠错的主动性。
