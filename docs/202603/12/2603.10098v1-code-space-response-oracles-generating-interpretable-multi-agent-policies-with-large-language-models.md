## 摘要
多智能体强化学习的最新进展，特别是策略空间响应预言机（PSRO），使得在日益复杂的领域中计算近似博弈论均衡成为可能。然而，这些方法依赖于深度强化学习预言机，其生成的“黑盒”神经网络策略难以解释、信任或调试。我们提出了代码空间响应预言机（CSRO），这是一个通过将强化学习（RL）预言机替换为大语言模型（LLM）来应对这一挑战的新型框架。CSRO 将最佳响应计算重新定义为代码生成任务，提示 LLM 直接生成人类可读的代码作为策略。这种方法不仅产生了具有内在可解释性的策略，还利用 LLM 的预训练知识来发现复杂的、类人的策略。我们探索了构建和增强基于 LLM 的预言机的多种方式：零样本提示、迭代细化以及 AlphaEvolve（一种基于 LLM 的分布式进化系统）。我们证明了 CSRO 在产生多样化可解释策略的同时，实现了与基准方法相当的性能。我们的工作为多智能体学习提供了一个新视角，将关注点从优化不透明的策略参数转向合成可解释的算法行为。

## Abstract
Recent advances in multi-agent reinforcement learning, particularly Policy-Space Response Oracles (PSRO), have enabled the computation of approximate game-theoretic equilibria in increasingly complex domains. However, these methods rely on deep reinforcement learning oracles that produce `black-box' neural network policies, making them difficult to interpret, trust or debug. We introduce Code-Space Response Oracles (CSRO), a novel framework that addresses this challenge by replacing RL oracles with Large Language Models (LLMs). CSRO reframes the best response computation as a code generation task, prompting an LLM to generate policies directly as human-readable code. This approach not only yields inherently interpretable policies but also leverages the LLM's pretrained knowledge to discover complex, human-like strategies. We explore multiple ways to construct and enhance an LLM-based oracle: zero-shot prompting, iterative refinement and \emph{AlphaEvolve}, a distributed LLM-based evolutionary system. We demonstrate that CSRO achieves performance competitive with baselines while producing a diverse set of explainable policies. Our work presents a new perspective on multi-agent learning, shifting the focus from optimizing opaque policy parameters to synthesizing interpretable algorithmic behavior.

### 1. 核心问题与研究动机
*   **核心问题**：传统的多智能体强化学习（MARL）算法（如 PSRO）依赖深度强化学习作为“预言机”（Oracle）来计算最优反应。这导致生成的策略是**不透明的“黑盒”神经网络**，难以被人类理解、验证或调试，且训练过程通常需要数百万次模拟，效率低下。
*   **研究背景**：在自动驾驶、网络安全等高风险领域，策略的可解释性和鲁棒性至关重要。现有的 LLM-PSRO 虽然尝试引入大模型，但缺乏主动优化机制，且受限于上下文窗口，难以处理复杂博弈。

### 2. 方法论：代码空间响应预言机 (CSRO)
*   **核心思想**：将寻找“最优反应”（Best Response）的过程从数值优化重构为**程序合成（Program Synthesis）**任务。利用 LLM 直接生成可执行、带注释的 Python 代码作为智能体策略。
*   **关键技术细节**：
    *   **代码策略（Code Policies）**：策略不再是权重参数，而是映射观察到动作的函数。
    *   **上下文抽象（Context Abstraction）**：为了应对长代码带来的上下文限制，CSRO 将对手策略的代码总结为自然语言描述，或仅筛选对均衡贡献最大的前 K 个对手。
    *   **三种预言机细化机制**：
        1.  **Zero-Shot**：直接根据博弈规则和对手描述生成代码。
        2.  **Linear Refinement**：通过“评估-反馈-修改”的闭环，在策略表现不佳时提示 LLM 进行迭代修复。
        3.  **AlphaEvolve**：集成分布式进化系统，通过多线程变异和筛选，在代码空间中搜索最优算法。
*   **算法流程**：遵循 PSRO 的迭代框架：计算当前元博弈均衡 -> 构造包含对手代码/描述的 Prompt -> LLM 生成新代码策略 -> 评估并加入策略池 -> 重复。

### 6. 主要结论与发现
*   **性能领先**：CSRO（尤其是 AlphaEvolve 版本）在 RRPS 和 Leduc 扑克中均显著优于深度强化学习基准（PSRO-IMPALA）。
*   **可利用性低**：AlphaEvolve 能够生成极低可利用性的策略，接近博弈论均衡。
*   **超强进化能力**：在 Leduc 扑克中，CSRO 发现的策略对特定对手（如 AlwaysCall）的剥削收益远超 CFR+，展现了“超纳什”水平的适应性。
*   **策略深度**：生成的代码展示了复杂的逻辑，如 RRPS 中的 32 个专家集成系统和 Leduc 中的期望价值（EV）计算。

### 7. 优点与亮点
*   **极致的可解释性**：策略以带注释的 Python 代码形式存在，人类可以直接阅读其逻辑（如“如果对手连续三次出石头，则...”），这在安全敏感领域具有巨大价值。
*   **知识迁移**：利用了 LLM 预训练中蕴含的博弈逻辑和编程能力，避免了从零开始的低效探索。
*   **计算经济性**：相比于每步推理的 LLM 智能体，CSRO 生成的“代码策略”在部署时几乎不消耗推理成本。

### 8. 不足与局限
*   **模型依赖性**：CSRO 的表现高度依赖于底层 LLM 的代码生成能力，若模型逻辑推理较弱，生成的策略可能存在语法错误或逻辑漏洞。
*   **可扩展性挑战**：对于观察空间极其复杂（如《星际争霸》或高维图像输入）的游戏，如何将状态抽象为 LLM 可理解的代码 API 仍是一个工程难题。
*   **Prompt 敏感性**：虽然有迭代细化，但初始 Prompt 的设计对最终收敛速度和质量仍有显著影响。
