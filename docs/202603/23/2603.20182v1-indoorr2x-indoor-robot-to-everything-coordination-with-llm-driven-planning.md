## 摘要
尽管机器人对机器人（R2R）通信提升了室内场景理解能力，超越了单机器人的局限，但若不增加大量的探索开销或扩大团队规模，仅靠 R2R 无法克服部分可观测性问题。相比之下，许多室内环境已经部署了低成本的物联网（IoT）传感器（如摄像头），它们能提供超越机载感知的、覆盖全建筑的持久上下文信息。因此，我们推出了 IndoorR2X，这是首个针对室内环境下具有机器人与万物（R2X）感知与通信能力的、由大语言模型（LLM）驱动的多机器人任务规划基准和仿真框架。IndoorR2X 整合了移动机器人和静态 IoT 设备的观测结果，以构建全局语义状态，支持可扩展的场景理解，减少冗余探索，并通过基于 LLM 的规划实现高层协同。IndoorR2X 提供了可配置的仿真环境、传感器布局、机器人团队和任务套件，用于系统地评估高层语义协同策略。在多种设置下的广泛实验表明，IoT 增强的世界建模提高了多机器人的效率和可靠性，我们还强调了推进机器人团队与室内 IoT 传感器之间基于 LLM 协作的关键见解和失败模式。

## Abstract
Although robot-to-robot (R2R) communication improves indoor scene understanding beyond what a single robot can achieve, R2R alone cannot overcome partial observability without substantial exploration overhead or scaling team size. In contrast, many indoor environments already include low-cost Internet of Things (IoT) sensors (e.g., cameras) that provide persistent, building-wide context beyond onboard perception. We therefore introduce IndoorR2X, the first benchmark and simulation framework for Large Language Model (LLM)-driven multi-robot task planning with Robot-to-Everything (R2X) perception and communication in indoor environments. IndoorR2X integrates observations from mobile robots and static IoT devices to construct a global semantic state that supports scalable scene understanding, reduces redundant exploration, and enables high-level coordination through LLM-based planning. IndoorR2X provides configurable simulation environments, sensor layouts, robot teams, and task suites to systematically evaluate high-level semantic coordination strategies. Extensive experiments across diverse settings demonstrate that IoT-augmented world modeling improves multi-robot efficiency and reliability, and we highlight key insights and failure modes for advancing LLM-based collaboration between robot teams and indoor IoT sensors.

### 2. 方法论：核心思想与技术细节
*   **核心思想**：建立一个中央“协调枢纽（Coordination Hub）”，将移动机器人的机载感知与静态 IoT 传感器的观测数据融合为统一的**全局语义状态（Global Semantic State）**，并利用 LLM 进行高层逻辑推理和并行任务分配。
*   **关键技术细节**：
    *   **全局语义状态更新**：状态 $W_t$ 包含机器人、物体和区域的属性。通过 transition 函数 $F$ 处理来自机器人和 IoT 设备的异步更新消息。
    *   **R2X 数据融合**：利用视觉语言模型（VLM，如 Qwen-VL）将 CCTV 视频流转化为带时间戳的文本事件日志，输入到协调枢纽。
    *   **LLM 驱动规划**：LLM 接收当前任务目标和全局语义状态，生成一个**有向无环图（DAG）**形式的计划，其中节点代表动作，边代表依赖关系。
    *   **并行执行与监测**：系统编排器根据 DAG 将任务分发给空闲机器人，并实时监测执行状态。若发生失败或 IoT 报告环境变化，则触发即时重规划。

### 6. 主要结论与发现
*   **效率提升**：R2X 相比 R2R 在保持高成功率（92%）的同时，显著减少了路径长度（约 11%）和动作步数，并降低了约 11% 的 LLM Token 成本。
*   **模型依赖性**：协同质量高度依赖 LLM 能力。GPT-4.1 表现优异，而 8B 规模的模型几乎无法完成复杂的协同任务。
*   **鲁棒性不对称**：系统对 **IoT 信息缺失**（漏检）具有极强的鲁棒性（成功率不变，仅增加探索成本）；但对 **IoT 语义错误**（如错误报告设备开关状态）非常敏感，会导致规划失败。
*   **扩展性**：团队规模增加会带来额外的协调开销（总路径长度增加），但在 5 台机器人以内成功率保持稳定。

### 7. 优点（亮点）
*   **范式创新**：首次将 R2X 概念系统化地引入室内多机协作，打破了以往研究仅关注机器人自身感知的局限。
*   **真实性约束**：基准测试严格执行局部可观测性，不给规划器提供“上帝视角”的 Oracle 信息，更符合实际应用。
*   **软硬件结合**：不仅有大规模仿真，还通过物理 Stretch 机器人进行了实地验证，证明了框架的可迁移性。

### 8. 不足与局限
*   **语义错误风险**：实验表明系统极度依赖 IoT 数据的准确性，缺乏对虚假/错误信息的验证机制。
*   **LLM 成本**：高性能规划依赖于 GPT-4 等昂贵的闭源模型，在实时性和经济性上对大规模部署提出挑战。
*   **动态环境复杂性**：虽然考虑了 IoT 延迟，但对于极高动态（如行人密集移动）的环境，语义状态的同步频率可能成为瓶颈。
*   **仿真偏差**：虽然有物理实验，但 85 个场景主要在仿真中完成，真实世界中 IoT 设备的安装位置、遮挡和网络波动可能比仿真更复杂。
