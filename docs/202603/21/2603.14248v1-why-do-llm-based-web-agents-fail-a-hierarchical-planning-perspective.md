## 摘要
基于大语言模型（LLM）的网络智能体越来越多地被用于网页导航，但在现实的长程任务中，其可靠性仍远低于人类。现有的评估主要关注端到端的成功率，对于失败发生的原因提供的见解有限。我们提出了一个分层规划框架，从三个层面（即高层规划、底层执行和重新规划）分析网络智能体，从而实现对推理、接地（grounding）和恢复能力的基于过程的评估。我们的实验表明，结构化的规划领域定义语言（PDDL）规划比自然语言（NL）规划能产生更简洁且目标导向的策略，但底层执行仍然是主要的瓶颈。这些结果表明，要达到人类水平的可靠性，不仅需要提高高层推理能力，改进感知接地和自适应控制也至关重要。这种分层视角为诊断和推进 LLM 网络智能体提供了原则性的基础。

## Abstract
Large language model (LLM) web agents are increasingly used for web navigation but remain far from human reliability on realistic, long-horizon tasks. Existing evaluations focus primarily on end-to-end success, offering limited insight into where failures arise. We propose a hierarchical planning framework to analyze web agents across three layers (i.e., high-level planning, low-level execution, and replanning), enabling process-based evaluation of reasoning, grounding, and recovery. Our experiments show that structured Planning Domain Definition Language (PDDL) plans produce more concise and goal-directed strategies than natural language (NL) plans, but low-level execution remains the dominant bottleneck. These results indicate that improving perceptual grounding and adaptive control, not only high-level reasoning, is critical for achieving human-level reliability. This hierarchical perspective provides a principled foundation for diagnosing and advancing LLM web agents.

#### 1. 核心问题与整体含义（研究动机和背景）
尽管大语言模型（LLM）在推理和指令遵循方面取得了显著进步，但在处理真实、动态网站的长程任务时，其可靠性仍远低于人类。现有的评估方法大多采用“端到端”的成功率指标，这虽然能量化整体表现，但无法揭示失败的具体来源：是任务理解错误、高层策略失误、底层 UI 操作无法落地，还是在环境变化时缺乏恢复能力？本论文旨在通过**分层规划（Hierarchical Planning）**的视角，将智能体能力拆解为高层规划、底层执行和重新规划三个维度，从而系统性地诊断 LLM 网络智能体的瓶颈。

#### 2. 方法论：核心思想与技术细节
论文提出了一个三层评估框架，模拟人类解决复杂任务的逻辑：
*   **高层规划（High-level Planning）**：将自然语言指令分解为一系列抽象子目标（Subgoals）。研究对比了两种表示形式：
    *   **自然语言（NL）**：灵活但往往过于冗长或缺乏结构。
    *   **PDDL（规划领域定义语言）**：一种结构化符号表示，强制要求明确的前置条件和效果。
*   **底层执行（Low-level Execution）**：将每个子目标转化为具体的网页交互动作（如点击、输入、滚动）。
*   **后置条件检查（Postcondition Checking）**：利用 LLM 作为评判者（LLM-as-Judge），验证当前动作序列是否真正达成了预设的子目标。
*   **重新规划（Replanning）**：当子目标执行失败时，智能体需决定是局部调整还是全局重写计划。

#### 3. 实验设计
*   **数据集与 Benchmark**：使用了 **Mind2Web-Live** 测试集（包含 104 个在真实动态网站上执行的任务），并基于专家标注的“关键节点”构建了人类基准计划。
*   **对比模型**：评估了三款主流模型：**gpt-5-nano**（主要模型）、**claude-haiku-4.5** 和 **gemini-flash-2.5**。
*   **动作空间设置**：对比了“扩展动作空间”（包含 `google_search` 等高级工具）和“原始动作空间”（仅限点击、输入等基础 UI 操作）。
*   **评估指标**：
    *   **对齐度指标**：完美匹配率、部分匹配率、缺失率、分解率等。
    *   **执行指标**：子目标完成率、计划完成率、任务成功率、动作冗余率等。

#### 5. 实验数量与充分性
*   **实验规模**：针对 104 个真实网页任务，在 3 种模型、2 种规划表示（NL/PDDL）、3 种动作空间表示下进行了多轮实验。
*   **充分性**：实验涵盖了从高层对齐到低层执行错误的深度分析，并进行了人工验证（抽样 50 组数据验证 LLM-as-Judge 的可靠性，一致性达 82%-86%），实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **PDDL 优于 NL**：使用 PDDL 引导的规划比自然语言更简洁、目标导向，且与人类意图的对齐度更高。
*   **底层执行是核心瓶颈**：即使给智能体提供完美的人类高层计划，其最终成功率依然较低（约 36.4%）。失败主要源于**感知接地（Perceptual Grounding）**不足，如幻觉链接、无法识别视觉相似元素、在无效操作上陷入死循环。
*   **重新规划至关重要**：仅需一轮基于环境反馈的重新规划，就能显著提升任务成功率，证明了闭环反馈的重要性。
*   **模型差异**：gpt-5-nano 表现最强；gemini-flash-2.5 在底层执行和环境理解上挣扎较多，冗余动作率最高。

#### 7. 优点（亮点）
*   **诊断性强**：不再只看“成败”，而是指出了“为什么败”，为后续研究提供了明确的改进方向（即加强底层控制而非仅仅堆叠推理能力）。
*   **引入符号化规划**：首次将 PDDL 引入 Web 智能体评估，证明了结构化约束对 LLM 规划的正面作用。
*   **细粒度指标**：提出了一套完整的对齐度评估体系（如分解率、冗余率），比单一的成功率更具参考价值。

#### 8. 不足与局限
*   **模态限制**：实验主要基于文本 DOM 表示，未涉及多模态（视觉）输入，这可能限制了对复杂视觉布局网站的分析。
*   **样本量较小**：104 个任务虽然来自真实环境，但对于全面覆盖互联网的多样性来说仍显不足。
*   **评估依赖性**：高度依赖 LLM-as-Judge 进行后置条件检查，尽管有人工抽检，但仍存在潜在的评估偏差。
*   **人工成本**：高层计划的对齐评估仍需要一定程度的人工标注或高质量参考计划，扩展性受限。
