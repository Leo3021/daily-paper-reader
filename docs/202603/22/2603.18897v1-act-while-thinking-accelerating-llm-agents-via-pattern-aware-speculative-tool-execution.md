## 摘要
以大语言模型（LLM）驱动的智能体正成为自主任务解决的主流范式。与标准的推理负载不同，智能体运行在严格串行的“LLM-工具”循环中，LLM 在每一步都必须等待外部工具的执行。这种执行模型引入了严重的延迟瓶颈。为解决这一问题，我们提出了 PASTE，一种模式感知的投机性工具执行方法，旨在通过投机执行来隐藏工具延迟。PASTE 基于以下洞察：尽管智能体请求在语义上具有多样性，但它们表现出稳定的应用级控制流（重复出现的工具调用序列）和可预测的数据依赖关系（工具间的参数传递）。通过利用这些特性，PASTE 通过投机性工具执行提升了智能体服务的性能。与最先进的基准方法相比，实验结果表明 PASTE 将平均任务完成时间缩短了 48.5%，并将工具执行吞吐量提高了 1.8 倍。

## Abstract
LLM-powered agents are emerging as a dominant paradigm for autonomous task solving. Unlike standard inference workloads, agents operate in a strictly serial "LLM-tool" loop, where the LLM must wait for external tool execution at every step. This execution model introduces severe latency bottlenecks. To address this problem, we propose PASTE, a Pattern-Aware Speculative Tool Execution method designed to hide tool latency through speculation. PASTE is based on the insight that although agent requests are semantically diverse, they exhibit stable application level control flows (recurring tool-call sequences) and predictable data dependencies (parameter passing between tools). By exploiting these properties, PASTE improves agent serving performance through speculative tool execution. Experimental results against state of the art baselines show that PASTE reduces average task completion time by 48.5% and improves tool execution throughput by 1.8x.

论文针对当前大语言模型（LLM）智能体在执行任务时存在的**严重延迟瓶颈**提出了解决方案。目前的智能体遵循严格串行的“推理-工具调用”循环（ReAct 模式）：LLM 生成指令 -> 等待工具执行 -> 获取结果 -> 继续推理。实验表明，工具执行时间占总请求时间的 35% 到 61%。由于这种串行依赖，LLM 在工具执行期间处于闲置状态，导致端到端延迟极高。论文的核心动机是利用智能体行为中存在的**可预测模式**，通过“投机性执行”（Speculative Execution）让工具在 LLM 思考时提前运行，从而掩盖工具执行的延迟。

#### 2. 方法论：核心思想与关键技术
论文提出了 **PASTE**（Pattern-Aware Speculative Tool Execution），其核心思想是将非确定性的智能体行为抽象为稳定的模式，并利用空闲资源进行风险可控的提前计算。

*   **模式元组（Pattern Tuple）**：定义为 `(context, prediction, function, probability)`。
    *   **Context (上下文)**：基于工具调用序列的签名（如 `Search` 成功后接 `Web_fetch`），排除自然语言的干扰。
    *   **Target (目标)**：预测下一步可能调用的工具。
    *   **Value Mapping Function (值映射函数)**：这是关键创新，它不靠 LLM 预测参数，而是通过符号化逻辑从历史输出中提取参数（例如：从搜索结果的 JSON 中提取第一个 URL 作为下载工具的参数）。
    *   **Probability (概率)**：基于历史数据的置信度。
*   **两阶段模式挖掘**：
    1.  **结构挖掘**：识别频繁出现的工具调用序列。
    2.  **符号依赖推断**：推断工具参数是如何从前序步骤的输出中派生出来的。
*   **机会主义调度器（Opportunistic Scheduler）**：
    *   **优先级管理**：正式的工具调用具有绝对优先级，投机任务仅利用“碎片资源”（Slack Resources）。

#### 7. 优点与亮点
*   **解耦控制流与数据流**：通过符号化映射函数解决了投机执行中最难的“参数预测”问题，避免了 LLM 重新生成参数带来的幻觉和延迟。
*   **非侵入式设计**：作为中间件实现，无需修改 LLM 内核或智能体逻辑。
*   **风险感知调度**：引入了严格的副作用控制策略和资源抢占机制，确保了投机失败时不会增加主任务的延迟，也不会产生不可逆的外部影响。

#### 8. 不足与局限
*   **副作用限制**：对于具有强副作用的操作（如删除文件、发送邮件），系统只能进行“干跑”（Dry-run）或环境预热，无法实现完全的执行加速。
*   **依赖历史模式**：如果智能体的任务极其冷门或完全随机，缺乏历史轨迹进行模式挖掘，PASTE 的增益将大幅下降。
*   **资源冗余**：虽然利用的是空闲资源，但在资源极度紧张的极端环境下，投机执行可能会因为频繁被抢占而失效。
*   **应用范围**：主要针对工具密集型智能体，对于纯推理型（Tool-less）任务没有帮助。
