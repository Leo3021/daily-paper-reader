## 摘要
计算机使用智能体（CUA）通过依赖视觉语言模型（VLM）来解释屏幕截图并预测定位的工具调用，从而将自然语言指令转换为图形用户界面（GUI）操作（如点击、按键和滚动）。然而，不同 VLM 的定位准确率差异巨大，而目前的 CUA 系统通常不考虑任务难度，将每个操作都路由到单个固定模型。我们提出了自适应 VLM 路由（AVR），这是一个在 CUA 编排器和 VLM 池之间插入轻量级语义路由层的框架。对于每个工具调用，AVR 通过多模态嵌入估计操作难度，探测小型 VLM 以测量置信度，并将操作路由到其预测准确率满足目标可靠性阈值的最廉价模型。对于具有先前 UI 交互记忆的“热”智能体，检索到的上下文进一步缩小了小型模型和大型模型之间的能力差距，使得许多操作无需升级即可处理。我们将路由形式化为成本与准确率的权衡，推导出基于阈值的模型选择策略，并使用 ScreenSpot-Pro 定位数据以及 OpenClaw 智能体路由基准对 AVR 进行评估。在这些设置中，AVR 预计可降低高达 78% 的推理成本，同时与全大模型基准相比，准确率差距保持在 2 个百分点以内。当与 Visual Confused Deputy 护栏结合使用时，AVR 还会将高风险操作直接升级到最强大的可用模型，从而在单一路由框架内实现了效率与安全性的统一。相关模型、基准和代码已提供：https://github.com/vllm-project/semantic-router。

## Abstract
Computer Use Agents (CUAs) translate natural-language instructions into Graphical User Interface (GUI) actions such as clicks, keystrokes, and scrolls by relying on a Vision-Language Model (VLM) to interpret screenshots and predict grounded tool calls. However, grounding accuracy varies dramatically across VLMs, while current CUA systems typically route every action to a single fixed model regardless of difficulty. We propose \textbf{Adaptive VLM Routing} (AVR), a framework that inserts a lightweight semantic routing layer between the CUA orchestrator and a pool of VLMs. For each tool call, AVR estimates action difficulty from multimodal embeddings, probes a small VLM to measure confidence, and routes the action to the cheapest model whose predicted accuracy satisfies a target reliability threshold. For \textit{warm} agents with memory of prior UI interactions, retrieved context further narrows the capability gap between small and large models, allowing many actions to be handled without escalation. We formalize routing as a cost--accuracy trade-off, derive a threshold-based policy for model selection, and evaluate AVR using ScreenSpot-Pro grounding data together with the OpenClaw agent routing benchmark. Across these settings, AVR projects inference cost reductions of up to 78\% while staying within 2 percentage points of an all-large-model baseline. When combined with the Visual Confused Deputy guardrail, AVR also escalates high-risk actions directly to the strongest available model, unifying efficiency and safety within a single routing framework. Materials are also provided Model, benchmark, and code: https://github.com/vllm-project/semantic-router.

### 论文总结：计算机使用智能体的自适应视觉语言模型路由 (AVR)

#### 1. 核心问题与整体含义（研究动机和背景）
随着计算机使用智能体（CUA）的发展，系统需要频繁调用视觉语言模型（VLM）来解析屏幕截图并执行点击、输入等操作。然而，当前系统面临两个核心痛点：
*   **高昂的推理成本**：每个任务通常需要 5-20 次迭代，每次携带大量多模态上下文（截图+历史），导致单次任务成本高达 $0.10–$0.40。
*   **模型规模与性能的不匹配**：实验发现，GUI 定位（Grounding）能力并不随参数量线性增长。例如，参数量巨大的 GPT-4o 在某些 GUI 基准上表现远逊于专门优化的 7B 模型。
*   **任务难度差异大**：简单的点击操作（如点击“提交”按钮）无需昂贵的顶级模型，而复杂的专业软件操作则需要更强的推理能力。

**研究动机**：开发一种自适应路由框架，动态选择最经济且能胜任当前步骤的模型，以平衡成本、精度与安全性。

#### 2. 方法论：核心思想与关键技术
论文提出了 **Adaptive VLM Routing (AVR)** 框架，其核心流程如下：

*   **多模态难度分类**：
    *   使用轻量级（120M参数）嵌入模型（如 SigLIP 和 MiniLM）提取截图区域和动作描述的特征。
    *   通过与已知“难/易”案例的知识库进行余弦相似度对比，计算视觉难度 $d_{vis}$ 和语义难度 $d_{sem}$，取最大值作为综合难度评分 $d(t_i)$。
*   **基于置信度的探测（Logprob Probing）**：
    *   先对小型 VLM（如 7B 模型）进行非流式探测，获取其输出 token 的平均对数概率（logprobs）作为置信度分数 $conf(t_i)$。
    *   **路由决策公式**：若 $conf(t_i) \geq \tau(\hat{d}_i)$，则接受小模型结果；否则升级（Escalate）至大模型。其中阈值 $\tau$ 随难度动态调整。
*   **记忆补偿机制（Memory-Compensated Routing）**：
    *   针对“热”智能体（有历史交互记录），检索相关的 UI 布局和操作经验注入 Prompt。
    *   **核心假设（记忆等化假设）**：记忆对小模型的能力提升远大于大模型，从而使小模型能处理更多原本需要升级的任务。
*   **安全集成**：
    *   集成“视觉受控代理”（Visual Confused Deputy）护栏，一旦检测到高风险动作（如删除、支付），强制升级至最强模型进行二次验证。

#### 5. 实验数量与充分性
*   **实验覆盖面**：涵盖了从通用大模型到 GUI 专用小模型的横向对比，并深入分析了 26 个不同专业软件（如 Photoshop, VS Code, Blender）的难度差异。

#### 6. 主要结论与发现
*   **成本大幅下降**：在保持与全大模型基准误差 2% 以内的前提下，AVR 预计可降低 **52%（冷启动）至 78%（热启动+难度分类）** 的推理成本。

*   **规模非唯一因素**：在 GUI 任务中，7B 的专业模型（如 OS-Atlas）表现可优于 1.8T 的通用模型（GPT-4o），这为路由策略提供了理论支撑。

#### 7. 优点
*   **经济性极佳**：为生产环境下的 CUA 大规模部署提供了切实可行的降本方案。
*   **架构透明**：作为 CUA 编排器与 VLM 之间的透明代理，易于集成到现有系统中。
*   **安全与效率统一**：首次将安全护栏（风险检测）作为路由升级的一个触发条件，实现了安全与成本的协同优化。

#### 8. 不足与局限
*   **缺乏端到端实测数据**：论文中的核心成本节省数据（Table 4）是基于现有基准的“投影”计算，而非在 OSWorld 等真实动态环境下的端到端运行统计。
