## 摘要
当多智能体系统产生错误或有害的回答时，如果执行日志和智能体标识符不可用，谁该负责？多智能体语言系统日益依赖于委派和迭代细化等结构化交互，然而最终输出往往掩盖了底层的交互拓扑和智能体贡献。我们引入了 IET（隐式执行追踪），这是一个独立于元数据的框架，能够直接从生成的文本中实现令牌级归因，并提供了一种用于交互拓扑重建的简单机制。在生成过程中，特定于智能体的密钥信号被嵌入到令牌分布中，将文本转化为仅能通过密钥检测的自描述执行轨迹。在检测阶段，一种感知转换的评分方法会识别智能体交接点并重建交互图。实验表明，IET 在保持生成质量的同时，能以高准确度恢复智能体分段和协作结构，从而为多智能体语言系统实现隐私保护审计。

## Abstract
When a multi-agent system produces an incorrect or harmful answer, who is accountable if execution logs and agent identifiers are unavailable? Multi-agent language systems increasingly rely on structured interactions such as delegation and iterative refinement, yet the final output often obscures the underlying interaction topology and agent contributions. We introduce IET (Implicit Execution Tracing), a metadata-independent framework that enables token-level attribution directly from generated text and a simple mechanism for interaction topology reconstruction. During generation, agent-specific keyed signals are embedded into the token distribution, transforming the text into a self-describing execution trace detectable only with a secret key. At detection time, a transition-aware scoring method identifies agent handover points and reconstructs the interaction graph. Experiments show that IET recovers agent segments and coordination structure with high accuracy while preserving generation quality, enabling privacy-preserving auditing for multi-agent language systems.

### 2. 方法论：隐式执行追踪（IET）
IET 的核心思想是将归因信息“编码”进文本生成的概率分布中，使其成为一种仅凭密钥可读的隐式水印。
*   **信号注入（生成阶段）**：
    *   为每个智能体分配唯一的密钥。在生成 Token 时，通过一个调制算子 $W$ 修改 Logits 分布。
    *   这种修改是统计学上的微小偏移，不影响文本语义，但会留下特定智能体的“指纹”。
*   **交互拓扑编码**：
    *   利用位掩码（Bitmask）将智能体间的交互关系（如 A 传给 B）编码进邻接矩阵，并随生成过程动态更新状态，确保后续生成的文本携带了之前的路径信息。
*   **归因与重建（检测阶段）**：
    *   **局部一致性评分**：使用滑动窗口计算文本各处与特定智能体密钥的匹配度。
    *   **竞争性变点检测（Change-Point Detection）**：通过累积和算法（CUSUM）识别评分波动的转折点，从而精准定位智能体交接的边界。
    *   **拓扑恢复**：通过解码嵌入的位掩码，重建智能体间的有向交互图。

*   **高精度归因**：IET 在所有拓扑结构下均实现了超过 94% 的 Token 级准确率和约 0.93 的 IoU，远超 LLM 基准（IoU 仅 0.1-0.2）。

### 7. 优点与亮点
*   **解耦元数据**：首次实现了不依赖外部日志的多智能体执行追踪，解决了文本“脱离环境”后的审计难题。
*   **隐私保护**：通过密钥控制检测权限，且支持对文本进行脱敏处理而不破坏追踪信号。
*   **系统级仪表化**：将水印技术从简单的“AI 识别”提升到了“系统架构审计”的高度。

### 8. 不足与局限
*   **生成控制权要求**：该方法要求必须在生成端部署 IET 插件。对于无法干预 Logits 分布的第三方闭源 API（如直接调用 OpenAI 官方接口且不开放 Logits 偏置），该方法无法实施。
*   **潜在的文本质量风险**：虽然实验称质量得以保留，但在极端多智能体（如数十个智能体）场景下，频繁切换密钥和叠加信号是否会累积干扰文本的自然度仍需进一步验证。
*   **计算开销**：检测阶段需要对每个 Token 位置进行多智能体密钥的统计评分，对于超长文本的实时审计可能存在计算延迟。
