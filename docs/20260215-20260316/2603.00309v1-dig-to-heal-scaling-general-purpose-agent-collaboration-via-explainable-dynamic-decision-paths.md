## 摘要
日益流行的智能体 AI (agentic AI) 范式有望利用多个通用大语言模型 (LLM) 智能体的力量来协作完成复杂任务。虽然许多智能体 AI 系统利用预定义的工作流或智能体角色来降低复杂性，但理想情况下，这些智能体应该是真正自主的，即使在协作智能体数量增加时也能实现涌现式协作 (emergent collaboration)。然而在实践中，这种非结构化交互可能导致冗余工作和级联故障，且难以解释或纠正。在这项工作中，我们研究了由通用 LLM 智能体组成的多智能体系统，这些智能体在没有预定义角色、控制流或通信限制的情况下运行，而是依靠涌现式协作来解决问题。我们引入了动态交互图 (DIG)，它将涌现式协作捕捉为智能体激活和交互的随时间演化的因果网络。DIG 首次使涌现式协作变得可观察且可解释，从而能够直接从智能体的协作路径中实时识别、解释和纠正由协作引起的错误模式。因此，DIG 填补了在理解通用 LLM 智能体如何在真正的智能体多智能体系统中共同解决问题方面的关键空白。项目网页见：https://happyeureka.github.io/dig。

## Abstract
The increasingly popular agentic AI paradigm promises to harness the power of multiple, general-purpose large language model (LLM) agents to collaboratively complete complex tasks. While many agentic AI systems utilize predefined workflows or agent roles in order to reduce complexity, ideally these agents would be truly autonomous, able to achieve emergent collaboration even as the number of collaborating agents increases. Yet in practice, such unstructured interactions can lead to redundant work and cascading failures that are difficult to interpret or correct. In this work, we study multi-agent systems composed of general-purpose LLM agents that operate without predefined roles, control flow, or communication constraints, relying instead on emergent collaboration to solve problems. We introduce the Dynamic Interaction Graph (DIG), which captures emergent collaboration as a time-evolving causal network of agent activations and interactions. DIG makes emergent collaboration observable and explainable for the first time, enabling real-time identification, explanation, and correction of collaboration-induced error patterns directly from agents' collaboration paths. Thus, DIG fills a critical gap in understanding how general LLM agents solve problems together in truly agentic multi-agent systems. The project webpage can be found at: https://happyeureka.github.io/dig.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）智能体（Agent）的普及，多智能体系统（MAS）正从“预定义工作流”向“通用自主协作”演进。然而，在没有预定义角色、控制流或通信限制的**涌现式协作（Emergent Collaboration）**中，智能体之间的非结构化交互极易导致冗余工作、死锁、早熟终止或级联故障。这些故障在复杂的异步交互中难以被追踪、解释和纠正。

本论文的核心动机是：**如何使通用智能体之间的动态协作过程变得可观测、可解释，并实现实时的自动修复？** 论文提出了“动态交互图（DIG）”框架，旨在填补理解和调试真正自主的多智能体系统之间的空白。

#### 2. 方法论：核心思想与关键技术
论文引入了 **动态交互图（Dynamic Interaction Graph, DIG）**，将协作过程建模为一个随时间演化的因果网络。

*   **核心思想**：将智能体的交互抽象为二分图结构，节点分为**激活节点（Activation Nodes）**（代表智能体的决策行为）和**事件节点（Event Nodes）**（代表消息或任务载体）。
*   **图重写算子（Graph Rewrite Operators）**：定义了五种规范的交互原语，用于描述智能体如何处理输入事件：
    1.  **Respond (R)**：消耗输入并生成新输出。
    2.  **Wait (W)**：保留缓冲区，暂不处理。
    3.  **Reroute (RR)**：将事件转发给其他智能体。
    4.  **Discard (D)**：丢弃冗余或无关事件。
    5.  **Submit (S)**：生成最终答案并终止。
*   **拓扑故障特征化**：DIG 建立了从图拓扑结构到协作错误模式的映射，识别了 7 种主要错误，包括：
    *   **终止类错误**：早熟终止（ET）、缺失终止（MC）、孤儿事件（OE）、死锁（DL）。
    *   **进度类警告**：过度重定向（ER）、跨谱系聚合（CLA）、重复子问题求解（RSP）。
*   **系统修复机制（System Healing）**：当 DIG 检测到上述拓扑异常时，系统会通过注入状态信息、重定向阻塞事件或创建系统级引导事件来实时干预，引导系统恢复正常。

#### 6. 主要结论与发现
*   **性能提升**：在困难任务中，MAS+DIG 显著降低了 RMSE，并保持了较高的有效输出率。

*   **效率优势**：相比于 LLM Judge 频繁调用大模型进行诊断带来的高昂 Token 开销和时间延迟，基于图拓扑的 DIG 检测机制几乎是瞬时的，且干预更精准。
*   **可解释性**：DIG 首次实现了对“涌现协作”过程的可视化追踪，使开发者能够清晰看到任务是如何被拆解、转发和聚合的。

#### 7. 优点
*   **协议无关性**：不依赖特定的智能体架构或通信协议，具有极强的通用性。
*   **实时性**：能够在故障发生瞬间通过拓扑特征捕获异常，而非事后归因。
*   **结构化解释**：将复杂的自然语言交互转化为清晰的图重写逻辑，为多智能体系统的调试提供了科学工具。

#### 8. 不足与局限
*   **领域覆盖有限**：虽然在计数和分类任务上表现良好，但在更具创造性或长程规划的任务（如开放式软件开发）上的表现尚待验证。

*   **运行时间开销**：虽然比 LLM Judge 快，但由于 DIG 会在检测到错误时延长系统运行以进行修复，其总运行时间有时会高于直接失败的 MAS-Only 系统。
