## 摘要
我们提出了经验证的多智能体编排（Verified Multi-Agent Orchestration, VMAO），这是一个通过验证驱动的迭代循环来协调专用 LLM 智能体的框架。针对复杂查询，我们的系统将其分解为子问题的有向无环图（DAG），通过特定领域的智能体并行执行，利用基于 LLM 的评估来验证结果的完整性，并进行自适应重规划以弥补差距。主要贡献包括：(1) 在子问题 DAG 上实现具有自动上下文传播的依赖感知并行执行；(2) 验证驱动的自适应重规划，将基于 LLM 的验证器作为编排层级的协调信号；以及 (3) 可配置的停止条件，用于平衡回答质量与资源消耗。在 25 个专家精选的市场研究查询中，与单智能体基准相比，VMAO 将回答完整性从 3.1 提升至 4.2，来源质量从 2.6 提升至 4.1（1-5 分制），证明了编排层级的验证是多智能体质量保证的有效机制。

## Abstract
We present Verified Multi-Agent Orchestration (VMAO), a framework that coordinates specialized LLM-based agents through a verification-driven iterative loop. Given a complex query, our system decomposes it into a directed acyclic graph (DAG) of sub-questions, executes them through domain-specific agents in parallel, verifies result completeness via LLM-based evaluation, and adaptively replans to address gaps. The key contributions are: (1) dependency-aware parallel execution over a DAG of sub-questions with automatic context propagation, (2) verification-driven adaptive replanning that uses an LLM-based verifier as an orchestration-level coordination signal, and (3) configurable stop conditions that balance answer quality against resource usage. On 25 expert-curated market research queries, VMAO improves answer completeness from 3.1 to 4.2 and source quality from 2.6 to 4.1 (1-5 scale) compared to a single-agent baseline, demonstrating that orchestration-level verification is an effective mechanism for multi-agent quality assurance.

*   **研究动机**：在处理复杂的跨领域查询（如市场研究）时，信息往往分散在异构来源中，需要多种专业知识进行合成。现有的多智能体框架（如辩论式或角色扮演式）虽然提升了推理能力，但缺乏结构化的任务分解、严谨的质量验证机制以及根据反馈进行自适应调整的能力。
*   **核心问题**：如何在无需人类持续监督的情况下，协调多个专业化智能体高效、准确地完成复杂任务，并确保最终输出的完整性和可靠性？

### 2. 方法论
VMAO 框架采用了 **“计划-执行-验证-重规划-合成”** 的五阶段迭代循环：
*   **计划（Plan）**：将复杂查询分解为子问题的**有向无环图（DAG）**。每个子问题包含 ID、智能体类型、依赖关系、优先级和验证标准。
*   **执行（Execute）**：根据 DAG 的依赖关系并行执行子任务。支持上下文传播，即下游任务可以获取上游任务的结果。
*   **验证（Verify）**：使用高阶 LLM 作为验证器，评估每个子问题的回答状态（完成/部分/未完成）、完整性得分（0-1）及缺失维度。
*   **重规划（Replan）**：基于验证反馈，决定是重试低分任务、引入新查询以填补空白，还是合并多次尝试的结果。
*   **合成（Synthesize）**：当满足停止条件（如完整性 > 80%、达到 Token 预算或最大迭代次数）时，进行分层合成，生成带有准确来源引用的最终报告。
*   **智能体分类**：将智能体分为三层：数据采集层（RAG、Web、金融、竞争对手）、分析层（推理、原始数据处理）和输出层（文档、可视化）。

### 3. 实验设计
*   **场景与数据集**：专注于**市场调研**领域，使用了 25 个由专家精选的复杂查询，涵盖性能分析、竞争情报、财务调查和战略评估四个类别。
*   **基准测试（Benchmark）**：
    *   **Single-Agent**：单个推理智能体访问所有工具。
    *   **Static Pipeline**：预定义的智能体顺序执行（RAG -> Web -> 金融 -> 分析 -> 合成），无验证和重规划。
*   **评估指标**：采用 1-5 分制评估**完整性（Completeness）**和**来源质量（Source Quality）**。评估由 LLM 裁判（Claude Opus 4.5）初步评分，再由人类专家审核调整。

### 5. 实验数量与充分性
*   **实验规模**：共进行了 25 组复杂查询实验。虽然样本量不算巨大，但每个查询涉及 8-12 个子问题和多次迭代，单次查询消耗 50-110 万个 Token，执行时长 10-20 分钟。
*   **充分性与客观性**：实验涵盖了不同复杂度的查询类别，并进行了基准对比。引入人类专家对 LLM 评分进行校准（调整了约 15% 的评分），增强了评估的客观性。但论文缺乏组件级的消融实验（例如单独测试验证机制与重规划机制的贡献）。

### 6. 主要结论与发现
*   **性能提升**：VMAO 在完整性上从 3.1 提升至 4.2（+35%），在来源质量上从 2.6 提升至 4.1（+58%）。
*   **验证的价值**：验证驱动的重规划在**开放式战略评估**任务中获益最大（完整性提升 53%），因为这类任务初始分解往往不全面。
*   **资源权衡**：高质量的输出伴随着更高的成本，VMAO 的 Token 消耗约为单智能体的 8.5 倍。

### 7. 优点
*   **结构化编排**：通过 DAG 明确了任务依赖，支持高效的并行执行。
*   **解耦验证**：将验证逻辑从智能体执行中分离出来，作为编排层的全局信号，提高了系统的鲁棒性。
*   **自适应性**：能够根据执行结果动态调整计划，而非死板地遵循预设流程。
*   **工业级设计**：引入了 Token 预算、超时控制和工具调用限制等生产环境必备的安全机制。

### 8. 不足与局限
*   **成本与延迟**：由于多次迭代和高阶模型验证，Token 成本和响应延迟显著增加，不适用于实时性要求高的场景。
*   **验证局限性**：验证器主要评估“完整性”而非“事实准确性”，无法完全排除幻觉或事实错误。
*   **模型依赖**：实验仅基于 Claude 系列模型，框架在其他模型家族（如 GPT 或开源模型）上的泛化能力尚未验证。
*   **样本量限制**：25 个查询的样本量较小，且未报告置信区间。
