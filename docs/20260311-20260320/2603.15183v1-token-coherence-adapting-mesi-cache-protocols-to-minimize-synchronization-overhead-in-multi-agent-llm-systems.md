## 摘要
多智能体 LLM 编排在朴素广播模式下会产生同步成本，其规模随智能体数量 (n)、步骤 (S) 和工件大小 (|D|) 呈 O(n x S x |D|) 增长——我将这一机制称为“广播诱导的三重乘法开销”。我认为这种病态现象是全状态重新广播的结构性残留，而非多智能体协作的固有属性。核心主张：LLM 多智能体系统中的同步成本爆炸可以形式化地精确映射到共享内存多处理器中的缓存一致性问题，且 MESI 协议的失效机制在极小结构修改下即可迁移至工件同步。我构建了工件一致性系统 (ACS) 并证明了 Token 一致性定理：当 S > n + W(d_i) 时，延迟失效（lazy invalidation）至少能将成本降低 S/(n + W(d_i))，从而将 O(n x S x |D|) 转化为 O((n + W) x |D|)。一个经 TLA+ 验证的协议在约 2,400 个探索状态中强制执行了单写者安全性、单调版本控制和有界陈旧性。在四种工作负载配置下的仿真结果显示，在 V=0.05 时 Token 节省率为 95.0% +/- 1.3%，V=0.10 时为 92.3% +/- 1.4%，V=0.25 时为 88.3% +/- 1.5%，V=0.50 时为 84.2% +/- 1.3%——均超过了该定理的保守下界。在 V=0.9 时，节省率仍维持在约 81%，这与预测的崩溃阈值相反。贡献：(1) 形式化的 MESI 到工件状态映射；(2) 作为节省下界的 Token 一致性定理；(3) 具有三个已证明不变量的 TLA+ 验证协议；(4) 解决了“始终读取”异议的条件工件访问语义特征化；(5) 通过薄适配层与 LangGraph、CrewAI 和 AutoGen 集成的参考 Python 实现。

## Abstract
Multi-agent LLM orchestration incurs synchronization costs scaling as O(n x S x |D|) in agents, steps, and artifact size under naive broadcast -- a regime I term broadcast-induced triply-multiplicative overhead. I argue this pathology is a structural residue of full-state rebroadcast, not an inherent property of multi-agent coordination.   The central claim: synchronization cost explosion in LLM multi-agent systems maps with formal precision onto the cache coherence problem in shared-memory multiprocessors, and MESI-protocol invalidation transfers to artifact synchronization under minimal structural modification.   I construct the Artifact Coherence System (ACS) and prove the Token Coherence Theorem: lazy invalidation attenuates cost by at least S/(n + W(d_i)) when S > n + W(d_i), converting O(n x S x |D|) to O((n + W) x |D|). A TLA+-verified protocol enforces single-writer safety, monotonic versioning, and bounded staleness across ~2,400 explored states.   Simulation across four workload configurations yields token savings of 95.0% +/- 1.3% at V=0.05, 92.3% +/- 1.4% at V=0.10, 88.3% +/- 1.5% at V=0.25, and 84.2% +/- 1.3% at V=0.50 -- each exceeding the theorem's conservative lower bounds. Savings of ~81% persist at V=0.9, contrary to the predicted collapse threshold.   Contributions: (1) formal MESI-to-artifact state mapping; (2) Token Coherence Theorem as savings lower bound; (3) TLA+-verified protocol with three proven invariants; (4) characterization of conditional artifact access semantics resolving the always-read objection; (5) reference Python implementation integrating with LangGraph, CrewAI, and AutoGen via thin adapter layers.

### 论文总结：Token Coherence (令牌一致性协议)

#### 1. 核心问题与整体含义（研究动机和背景）
在当前的多智能体大语言模型（MAS）编排中，系统通常采用“全量广播”模式：每当一个共享工件（如计划文档、代码库）被修改，编排器会在下一个同步点将完整工件重新发送给所有订阅智能体。这导致同步成本随智能体数量 ($n$)、推理步骤 ($S$) 和工件大小 ($|D|$) 呈 $O(n \times S \times |D|)$ 的爆炸式增长。作者认为这种高昂的 Token 成本并非多智能体协作固有，而是早期架构设计的残留。论文提出将计算机体系结构中成熟的 **MESI 缓存一致性协议** 引入 MAS，以解决这一“同步病态”问题。

#### 2. 方法论：核心思想与关键技术
*   **核心思想**：将智能体类比为处理器核心，将共享工件类比为缓存行，通过追踪工件状态并仅在必要时发送失效信号（而非全量数据）来降低开销。
*   **工件一致性系统 (ACS)**：构建了一个六元组模型 $\langle A, D, \Sigma, \delta, \alpha, T \rangle$，将硬件 MESI 状态映射到工件授权状态：
    *   **M (Modified)**：智能体持有唯一有效副本，权威副本已过时。
    *   **E (Exclusive)**：智能体持有唯一副本且与权威副本一致。
    *   **S (Shared)**：多个智能体持有有效副本。
    *   **I (Invalid)**：副本已失效，下次使用需重新获取。
*   **关键技术**：
    *   **延迟失效 (Lazy Invalidation)**：仅在写入提交时触发失效，允许在多次写入间合并重新获取操作。
    *   **Token 一致性定理**：证明了节省下界，将成本从三重乘法关系转化为加法乘法关系 $O((n + W) \times |D|)$，其中 $W$ 为写入频率。
    *   **TLA+ 形式化验证**：验证了单写者安全 (SWMR)、单调版本化和 K-有界陈旧性（智能体不会基于过时状态推理超过 K 步）。

*   **Benchmark**：以主流框架默认的“全量广播同步”作为基准。
*   **对比方法**：对比了四种策略：广播基准、急切失效 (Eager)、延迟失效 (Lazy) 和基于访问计数的失效。
*   **评估维度**：Token 节省率、缓存命中率 (CHR) 以及随智能体数量、工件大小、步骤数的扩展性。

#### 6. 主要结论与发现
*   **显著节省**：在不同变动率下，Token 节省率达到 **84.2% 至 95.0%**。
*   **打破崩溃预测**：理论预测在变动率 $V \approx 0.9$ 时节省效果会消失，但仿真显示由于延迟合并机制，即使在极高变动率下仍能保持约 **81%** 的节省。
*   **扩展性优异**：随着步骤数 $S$ 增加，广播成本线性增长，而一致性协议成本增长极慢，验证了其消除 $S$ 乘数的能力。
*   **安全性**：形式化验证证明，失效机制是保证单写者安全的必要条件，而非仅仅是优化。

#### 7. 优点
*   **跨学科迁移**：成功将成熟的硬件协议理论应用于新兴的 LLM 编排领域，理论根基深厚。
*   **形式化严谨**：不仅有数学定理证明，还通过 TLA+ 进行了状态空间搜索验证。
*   **高兼容性**：通过薄适配层即可集成到 LangGraph、CrewAI 和 AutoGen 等主流框架，无需修改框架内核。
*   **实用性强**：直接针对生产环境中的 Token 成本痛点，提供了可量化的节省预期。

#### 8. 不足与局限
*   **仿真局限**：实验基于仿真，未完全捕捉真实生产环境中的网络延迟、LLM 推理抖动及复杂的异步并发行为。
*   **中心化瓶颈**：依赖单一的权威服务（Authority Service），在大规模分布式部署中可能成为性能瓶颈。

*   **策略匹配**：在“指针语义”（按需频繁获取小片段）场景下，延迟策略可能因频繁冷启动导致开销反超急切策略。
