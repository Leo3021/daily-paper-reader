## 摘要
利用大语言模型（LLM）进行机器人操作的任务规划是一个新兴领域。先前的方法依赖于专用模型、微调或提示调优，且通常以开环方式运行，缺乏稳健的环境反馈，这使得它们在动态环境中表现脆弱。我们提出了 MALLVi，这是一个多智能体大语言与视觉框架，可实现闭环反馈驱动的机器人操作。给定自然语言指令和环境图像，MALLVi 为机器人机械臂生成可执行的原子动作。动作执行后，视觉语言模型（VLM）评估环境反馈，并决定是重复该过程还是进行下一步。MALLVi 并非使用单一模型，而是协调专门的智能体——分解器（Decomposer）、定位器（Localizer）、思考器（Thinker）和反射器（Reflector），以管理感知、定位、推理和高层规划。一个可选的描述器（Descriptor）智能体提供初始状态的视觉记忆。反射器通过仅重新激活相关的智能体来支持有针对性的错误检测和恢复，从而避免了完全的重新规划。在仿真和真实世界环境中的实验表明，迭代闭环多智能体协调提高了泛化能力，并提升了零样本操作任务的成功率。代码可在 https://github.com/iman1234ahmadi/MALLVI 获取。

## Abstract
Task planning for robotic manipulation with large language models (LLMs) is an emerging area. Prior approaches rely on specialized models, fine tuning, or prompt tuning, and often operate in an open loop manner without robust environmental feedback, making them fragile in dynamic settings.We present MALLVi, a Multi Agent Large Language and Vision framework that enables closed loop feedback driven robotic manipulation. Given a natural language instruction and an image of the environment, MALLVi generates executable atomic actions for a robot manipulator. After action execution, a Vision Language Model (VLM) evaluates environmental feedback and decides whether to repeat the process or proceed to the next step.Rather than using a single model, MALLVi coordinates specialized agents, Decomposer, Localizer, Thinker, and Reflector, to manage perception, localization, reasoning, and high level planning. An optional Descriptor agent provides visual memory of the initial state. The Reflector supports targeted error detection and recovery by reactivating only relevant agents, avoiding full replanning.Experiments in simulation and real world settings show that iterative closed loop multi agent coordination improves generalization and increases success rates in zero shot manipulation tasks.Code available at https://github.com/iman1234ahmadi/MALLVI.

### 2. 方法论：核心思想与技术细节
MALLVi 的核心思想是将复杂的机器人操作任务拆解为多个专门智能体的协作流程，并引入**反射机制**实现闭环控制。
*   **多智能体架构**：
    1.  **Decomposer（分解器）**：将高层指令分解为原子动作序列（如 move, reach, push），并标注内存标签。
    2.  **Descriptor（描述器）**：利用 VLM 生成环境的粗略表示，构建空间图（Spatial Graph），提供视觉记忆。
    3.  **Localizer（定位器）**：包含 Perceptor（识别目标）、Grounder（利用 GroundingDINO 等进行 2D 定位）和 Projector（利用 SAM 提取抓取点并投影至 3D 空间）。
    4.  **Thinker（思考器）**：将子任务信息转化为具体的执行参数（如 3D 坐标、旋转角度）。
    5.  **Actor（执行器）**：通过 API 调用机器人底层控制完成动作。
    6.  **Reflector（反射器）**：核心组件，通过视觉反馈评估子任务是否成功。若失败，则仅激活相关智能体进行局部重试，而非全局重新规划。
*   **技术细节**：
    *   **3D 投影**：利用针孔相机模型和双目视觉原理，将图像坐标转换为机器人坐标系下的 3D 坐标。
    *   **抓取点优化**：使用 SAM 生成掩码，并通过几何启发式算法计算最优抓取位置，避免直接使用边界框中心点导致的抓取失败。

*   **自纠错能力**：Reflector 能够提供自然语言形式的失败原因分析，并触发局部修复。
*   **视觉记忆集成**：Descriptor 引入的空间图为后续步骤提供了稳定的环境上下文参考。
*   **开源友好**：代码已开源，且框架支持替换不同的 LLM/VLM 后端。

### 8. 不足与局限
*   **底层控制依赖**：框架仍依赖于预定义的原子动作（Atomic Actions），如果机器人遇到未预见的运动学约束或复杂的接触动力学，高层规划可能无法解决底层执行问题。
*   **延迟问题**：多智能体之间多次调用 LLM/VLM API 会带来显著的推理延迟，难以应对极高实时性的任务。
*   **感知局限**：在处理高度动态、纹理极其复杂或透明物体时，现有的视觉定位模块（如 GroundingDINO）可能仍存在瓶颈。
*   **成本风险**：频繁调用闭环反馈中的商业模型 API（如 GPT-4）可能产生较高的运行成本。
